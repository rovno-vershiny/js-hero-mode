# 1пр Клон библиотеки axios на минималках

## Задача
Создать класс `VershinyAxios`, который позволит:
1.  Создавать экземпляры с "базовыми настройками" (например, базовый URL API).
2.  Автоматически отправлять заголовки (headers) и превращать данные в JSON.
3.  Делать запросы короткими методами: `.get()`, `.post()`, `.put()`, `.delete()`.

---

## Техническое задание (ТЗ)

Вам нужно реализовать класс `VershinyAxios` со следующим функционалом:

1.  **Конструктор (`constructor`)**:
    *   Принимает объект настроек `config`.
    *   В настройках обычно передается `baseUrl` (например, `https://jsonplaceholder.typicode.com`) и `headers`.
    *   Сохраняет эти настройки в `this`.

2.  **Приватный метод запроса (внутренняя логика)**:
    *   Это сердце вашей библиотеки. Он должен объединять `baseUrl` с конкретным путем (endpoint).
    *   Он должен вызывать стандартный `fetch`.
    *   **Важно:** Если метод запроса `POST` или `PUT`, он должен автоматически добавлять заголовок `Content-Type: application/json` и превращать тело запроса (`body`) в строку через `JSON.stringify()`.
    *   Он должен возвращать **сразу данные** (то, что лежит в `response.json()`), а не сам объект ответа fetch.

3.  **Публичные методы**:
    *   `get(url)` — получение данных.
    *   `post(url, data)` — отправка данных (создание).
    *   `put(url, data)` — обновление данных (в ТЗ это "update").
    *   `delete(url)` — удаление данных.

4.  **Статический метод `create(config)`**:
    *   Должен возвращать **новый экземпляр** класса `VershinyAxios` с переданными настройками.

---

## Важные моменты

1. **Зачем нужен `static create`?**
    Это паттерн "Фабричный метод". Он позволяет писать красиво: `const api = VershinyAxios.create(...)` вместо `const api = new VershinyAxios(...)`. В `axios` принято делать именно так.

2. **Склеивание URL:**
    Будьте внимательны со слешами `/`. Если `baseUrl` заканчивается на `/`, а `endpoint` начинается на `/`, получится двойной слеш (`//`).

3. **JSON:**
    `fetch` не отправляет JSON сам по себе. Вы *обязаны* указать заголовок `'Content-Type': 'application/json'`, иначе сервер подумает, что вы прислали ему обычный текст, и не сможет прочитать объект.
